Tipuri de moștenire în POO C#

Tot despre tipurile de moștenire în programarea orientată pe obiecte, cu exemple de cod în C#.

Definiție

Ce este moștenirea?

Moștenirea este unul dintre pilonii fundamentali ai programării orientate pe obiecte. Permite unei clase (clasă derivată) să preia atribute și metode de la o altă clasă (clasă de bază), promovând reutilizarea codului și ierarhizarea logică a entităților.

Tipuri de moștenire

Tip 01

Moștenire simplă

O clasă derivată moștenește de la o singură clasă de bază. Este cel mai comun tip, suportat direct în C# prin cuvântul cheie :.

Tip 02

Moștenire multiplă (prin interfețe)

C# nu permite moștenirea multiplă de la clase, dar o clasă poate implementa mai multe interfețe simultan, obținând un comportament similar.

Tip 03

Moștenire pe niveluri

O clasă derivată devine, la rândul ei, clasă de bază pentru o altă clasă. Formează un lanț ierarhic: A → B → C.

Tip 04

Moștenire ierarhică

Mai multe clase derivate moștenesc de la aceeași clasă de bază. Un exemplu clasic: Animal → Câine, Pisică, Pasăre.

Tip 05

Clase abstracte

O clasă marcată cu abstract nu poate fi instanțiată direct. Definește un contract pe care clasele derivate trebuie să-l implementeze.

Tip 06

Clase sealed

O clasă marcată cu sealed nu mai poate fi moștenită. Folosită când se dorește prevenirea extinderii ierarhiei.

Cuvântul cheie base

Concept

La ce servește base?

Cuvântul cheie base permite clasei derivate să acceseze membrii (constructori, metode, proprietăți) clasei de bază. Este esențial atunci când clasa derivată suprascrie o metodă, dar dorește totuși să păstreze și comportamentul original al clasei de bază.

C# · Cuvântul cheie base 
public class Vehicul
{
    public string Marca { get; set; }

    public Vehicul(string marca)
    {
        Marca = marca;
        Console.WriteLine($"Vehicul creat: {Marca}");
    }

    public virtual void Descrie()
    {
        Console.WriteLine($"Vehicul: {Marca}");
    }
}

public class Masina : Vehicul
{
    public int NrUsi { get; set; }

    // Apelăm constructorul clasei de bază cu base()
    public Masina(string marca, int nrUsi)
        : base(marca)
    {
        NrUsi = nrUsi;
    }

    public override void Descrie()
    {
        // Apelăm metoda din clasa de bază, apoi adăugăm info extra
        base.Descrie();
        Console.WriteLine($"Număr uși: {NrUsi}");
    }
}

Polimorfismul în moștenire

Concept

Ce este polimorfismul?

Polimorfismul (din gr. „multe forme") permite ca un obiect al unei clase derivate să fie tratat ca un obiect al clasei de bază. Combinat cu metodele virtual și override, fiecare clasă derivată poate răspunde diferit la același apel de metodă.

C# · Polimorfism cu virtual / override 
public class Forma
{
    public virtual void Deseneaza()
        => Console.WriteLine("Desenez o formă generică.");
}

public class Cerc : Forma
{
    public override void Deseneaza()
        => Console.WriteLine("Desenez un cerc.");
}

public class Dreptunghi : Forma
{
    public override void Deseneaza()
        => Console.WriteLine("Desenez un dreptunghi.");
}

// Polimorfism în acțiune
Forma[] forme = { new Cerc(), new Dreptunghi(), new Forma() };

foreach (var f in forme)
    f.Deseneaza();

// Output:
// Desenez un cerc.
// Desenez un dreptunghi.
// Desenez o formă generică.

Moștenire hibridă

Concept avansat

Combinarea tipurilor de moștenire

Moștenirea hibridă combină mai multe tipuri de moștenire (de ex. ierarhică + pe niveluri) în aceeași aplicație. În C# se realizează prin combinarea claselor și interfețelor, creând ierarhii complexe, dar organizate.

C# · Moștenire hibridă (ierarhică + pe niveluri) 
// Nivelul 1 — clasă de bază
public class Angajat
{
    public string Nume { get; set; }
    public virtual void Rol() => Console.WriteLine("Angajat general");
}

// Nivelul 2 — moștenire pe niveluri
public class Manager : Angajat
{
    public override void Rol() => Console.WriteLine("Manager");
}

// Nivelul 3 — tot pe niveluri
public class Director : Manager
{
    public override void Rol() => Console.WriteLine("Director");
}

// Moștenire ierarhică — ambele din Angajat
public class Programator : Angajat
{
    public override void Rol() => Console.WriteLine("Programator");
}

Exemplu — Moștenire simplă

C# · Moștenire simplă 
// Clasa de bază
public class Animal
{
    public string Nume { get; set; }

    public virtual void Sunet()
    {
        Console.WriteLine("Animalul face un sunet.");
    }
}

// Clasa derivată
public class Caine : Animal
{
    public override void Sunet()
    {
        Console.WriteLine("Câinele latră: Ham! Ham!");
    }
}

Exemplu — Moștenire multiplă prin interfețe

C# · Interfețe multiple 
public interface INotabil
{
    void AfiseazaNota();
}

public interface IPrezentabil
{
    void Prezinta();
}

// Clasa implementează ambele interfețe
public class Elev : INotabil, IPrezentabil
{
    public void AfiseazaNota()  => Console.WriteLine("Nota: 10");
    public void Prezinta()      => Console.WriteLine("Mă numesc Cristian.");
}

Exemplu — Clasă abstractă

C# · Abstract 
public abstract class Forma
{
    // Metoda abstractă — fără implementare
    public abstract double Arie();

    // Metodă concretă — cu implementare
    public void Descrie()
    {
        Console.WriteLine($"Aria formei este: {Arie()}");
    }
}

public class Cerc : Forma
{
    private double raza;

    public Cerc(double raza) => this.raza = raza;

    public override double Arie() => Math.PI * raza * raza;
}

Greșeli frecvente

⚠ 01

Uitarea cuvântului override

Dacă o metodă virtuală nu este marcată cu override în clasa derivată, ea ascunde metoda de bază în loc să o suprascrie — comportament diferit și periculos.

⚠ 02

Instanțierea unei clase abstracte

Clasele abstracte nu pot fi instanțiate direct (new Forma() va produce eroare de compilare). Ele servesc doar ca bază pentru alte clase.

⚠ 03

Moștenire excesivă

Ierarhii prea adânci fac codul greu de înțeles și întreținut. Regula generală: dacă ai mai mult de 3–4 niveluri, reconsideră designul și folosește compoziția în locul moștenirii.
